Comprendre les métadonnées du livre

Qu'est-ce que le format ONIX 3.0 ?

Si vous vous lancez dans l'édition ou l'auto-édition professionnelle, vous rencontrerez inévitablement le terme ONIX. Loin d'être un simple caprice technique, c'est le langage universel qui permet à votre livre d'exister en librairie.

« ONIX est au livre ce que le code-barres est à la grande distribution : une carte d'identité numérique standardisée, indispensable pour automatiser les ventes. »

A quoi ça sert ?

Lorsqu'un livre sort, les libraires, les plateformes en ligne (Fnac, Amazon) et les grandes bases de données (Dilicom, Electre, Decitre) ont besoin de connaître ses informations (le prix, le format, le résumé, l'auteur). C'est ce qu'on appelle les métadonnées.

Sans ONIX (L'enfer des fichiers)

Vous devriez remplir un fichier Excel différent pour Dilicom, un autre pour Amazon, envoyer un mail à la Fnac, et copier-coller manuellement votre résumé partout. Risque d'erreurs maximal.

Avec ONIX (La synchronisation)

Vous générez un seul fichier unique au format .xml. Vous l'envoyez à votre distributeur. En quelques heures, toutes les bases de données du monde francophone se mettent à jour fidèlement.

Pourquoi la version 3.0 ?

La norme ONIX est gérée à l'échelle internationale par l'organisme EDItEUR. La version 2.1 est officiellement obsolète depuis de nombreuses années. La version ONIX 3.0 est aujourd'hui la seule et unique norme acceptée pour les nouveautés, car elle gère nativement et de façon ultra-précise :

  • Les subtilités du livre numérique (formats ePub, PDF, protections DRM).
  • Les marchés multiples (prix différents selon les pays).
  • La classification internationale Thema (qui remplace progressivement la vieille classification CLIL).

Comprendre les codes utilisés dans OniXpress

Pour qu'un ordinateur comprenne votre fichier sans ambiguïté linguistique, ONIX n'écrit pas "Livre Broché" ou "Disponible", il utilise des listes de codes codifiées (Codelists). Voici les équivalences de ce que le générateur OniXpress configure en arrière-plan :

Formats de produit (Liste 150)

Code ONIX Signification Description technique
BC Broché Couverture souple ordinaire (Paperback).
BB Relié Couverture rigide, cartonné (Hardback).
ED Livre numérique Fichier dématérialisé (ePub, PDF ou application).

Statuts de disponibilité (Liste 65)

Code ONIX Disponibilité commerciale Usage type
10 À paraître Le livre est annoncé, les libraires peuvent enregistrer des précommandes.
20 Disponible Le livre est en stock chez le distributeur et livrable immédiatement.
31 Épuisé Le livre ne sera plus réimprimé, la fiche est conservée pour historique.
40 Disponibilité incertaine En attente de réimpression ou rupture temporaire.

Comment utiliser votre fichier exporté ?

  1. Complétez l'intégralité des champs obligatoires marqués d'un astérisque rouge (*) sur le formulaire.
  2. Vérifiez visuellement l'épreuve à droite de votre écran.
  3. Cliquez sur "Télécharger le .xml".
  4. Déposez ou envoyez ce fichier sur l'espace professionnel de votre distributeur (Dilicom, plateformes d'auto-édition, ou outil de gestion interne).
