La philosophie derrière l'outil

Pourquoi avoir créé KartoLibr ?

Le format ONIX 3.0 est indispensable pour diffuser ses livres en librairie. Pourtant, pour les petites structures, l'accès à ce format relève souvent du parcours du combattant : entre les logiciels professionnels hors de prix et l'écriture manuelle d'un code XML ultra-complexe, beaucoup se retrouvent bloqués.

KartoLibr est né pour briser cette barrière. C'est un outil pensé par des amoureux du livre pour simplifier la vie de ceux qui les font.

100% Gratuit & Libre

Pas d'abonnement caché, pas de version premium, pas d'inscription obligatoire. L'outil est mis à disposition gratuitement et son code source est ouvert à la communauté (Open Source).

Pensez livre, pas code

Vous n'avez pas à savoir ce qu'est une balise XML, ni à mémoriser les listes de codes de la norme internationale. Vous remplissez un formulaire classique, KartoLibr s'occupe de la traduction technique.

Une priorité : les petites maisons d'édition

Les grands groupes éditoriaux possèdent des systèmes de gestion (ERP) automatisés capables d'exporter des flux de données complexes. Les éditeurs indépendants, les micro-structures et les auteurs en auto-édition, eux, gèrent souvent tout à la main.

Acheter une licence logicielle lourde pour seulement quelques sorties par an n'a pas de sens économique. KartoLibr leur offre une alternative professionnelle, conforme aux exigences de Dilicom et d'EDItEUR, sans dépenser un seul centime.

Comment utiliser l'outil ? (Guide pas à pas)

La création de votre métadonnée se fait en quelques minutes seulement. Voici la marche à suivre :

  • 01
    Saisir les informations du livre
    Étape obligatoire

    Remplissez les champs du formulaire à gauche de l'écran. Prenez un soin particulier à l'ISBN (sans tirets), au prix TTC et au résumé. Les champs indispensables sont signalés par un astérisque rouge.

  • 02
    Vérifier l'aperçu en temps réel
    Contrôle visuel

    À droite de votre écran, le volet d'aperçu génère le code ONIX 3.0 au fur et à mesure de votre saisie. C'est l'assurance visuelle que vos données se structurent correctement sous le capot.

  • 03
    Générer et télécharger le fichier
    Action

    Une fois le formulaire validé, cliquez sur le bouton de téléchargement. Votre navigateur va récupérer un fichier nommé onix_ISBN.xml. Ne le renommez pas avec une extension étrange, le format .xml est indispensable.

  • 04
    Transmettre à votre distributeur
    Diffusion

    Connectez-vous sur votre plateforme habituelle (Dilicom, Electre, ou l'espace de dépôt de votre distributeur) et chargez votre fichier XML tout frais. Les robots de traitement liront l'arborescence KartoLibr instantanément.

Souveraineté des données & Sécurité

Nous savons que vos programmes de parution et vos données financières sont sensibles. C'est pourquoi KartoLibr utilise une architecture dite "Client-Side" :

  • Zéro serveur : Tout s'exécute directement à l'intérieur de votre propre navigateur web.
  • Zéro stockage : Rien n'est enregistré sur un cloud externe. Fiche sauvegardée automatiquement dans votre navigateur (localStorage) — fermez l'onglet sans crainte, vous la retrouverez en revenant ici.
  • Confidentialité totale : Vos données ne quittent jamais votre machine. Vous pouvez même utiliser l'outil totalement déconnecté d'Internet.
